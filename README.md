# Hello May 喝樂梅｜苗栗頭屋鄉地方品牌與果酒體驗單頁式電商網站

本專案以 HTML、CSS、JavaScript 製作，可直接部署至 GitHub Pages。網站主題為「苗栗頭屋鄉 Hello May 喝樂梅地方品牌與果酒體驗」，目標是把地方果酒、禮盒詢價、社群信任與品牌故事整合成一個手機版優先的單頁式電商入口。

> 提醒：本網站含酒類資訊，已設計 18 歲 Age Gate。正式上線時仍需依實際法規、品牌資料與商品銷售規範調整。

---

## 1. Problem Statement

目前的挑戰是：由於地方酒類品牌網站常缺乏清楚的購買路徑、社群證言不足，且文案容易呈現制式 AI 感，導致消費者在想購買苗栗梅酒或酒類禮盒時無法快速建立信任與完成詢價，若持續下去將會降低轉換率、削弱品牌記憶點，並使地方品牌錯失年輕族群與商務送禮市場的電商成長機會。

本網站的電商解法是：以單頁式導購架構整合 Age Gate、品牌故事、線上詢價流程、社群牆、LINE CTA、禮盒諮詢表單與 FAQ，降低使用者在手機端的決策成本，讓「看見品牌、理解價值、產生信任、完成詢價」在同一頁完成。

## 2. 目標受眾 Persona 與 SEO 策略

- 需求：想找有特色、好拍、適合輕鬆分享的果酒
- 痛點：不想看太硬的規格文案，也不想在網站裡找不到購買入口
- 對應關鍵字：苗栗梅酒推薦、Hello May 喝樂梅、茶梅酒、微醺 Me Time

### Persona B：尋找儀式感的上班族
- 年齡：25–40 歲
- 情境：下班後想放鬆，週末想準備一瓶有質感的酒
- 需求：希望酒款有風味、有故事，也能作為送禮選擇
- 痛點：怕買到包裝普通、缺乏地方特色或不夠體面的禮盒
- 對應關鍵字：酒類禮盒推薦、苗栗在地酒廠、苗栗頭屋名產、苗栗梅酒推薦

## 3. 網站區塊邏輯：8 個區塊為何存在？

1. Age Gate 年齡阻擋視窗  
   解決酒類網站的基本合規問題，提醒未滿 18 歲不得飲酒，也讓網站在期末報告中具備倫理與法規意識。

2. Header 導覽列  
   解決手機使用者容易迷路的問題，保留品牌、故事、流程、表單與 LINE CTA 的快速入口。

3. 首頁主視覺區塊  
   用 H1 直接放入「Hello May 喝樂梅」，並以「下班後的微醺儀式」吸引 Z 世代與上班族，快速建立情緒情境。

4. 品牌故事與商品價值  
   將「苗栗頭屋名產」、「苗栗在地酒廠」、「茶香」、「職人低溫發酵」轉化為品牌價值，避免只賣商品，改成賣地方記憶與生活感。正式報告中若使用「金牌」等獎項字眼，需補上可查證來源或品牌授權資料。

5. 三步驟線上詢價流程圖  
   回應痛點「找不到結帳入口」，以選擇需求、LINE 專人回覆、完成訂購三步驟降低購買焦慮。

6. 微醺生活社群牆  
   回應痛點「缺乏社群證言」，用 KOL IG/FB/Reels 推薦截圖區建立信任。期末展示版為版位示意，正式版需替換成真實授權截圖或可公開引用之合作貼文。

7. 酒類禮盒推薦諮詢表單  
   解決使用者不知道該買哪一款的問題，透過用途、預算、數量與補充需求蒐集線索，適合商務饋贈與節慶送禮。

8. FAQ 常見問題  
   解決購買前疑慮，例如是否適合送禮、能否直接結帳、酒感是否太重，以及未滿 18 歲不可購買等問題。

補充微調：首頁保留原版型，但在品牌故事下新增「五款茶梅酒風味速覽」，讓輪播不只是展示瓶身，也能快速說明五款酒的差異，避免答辯時被質疑「商品只有照片、沒有選購理由」。

## 4. GA4 追蹤規劃

HTML 已預留 GA4 gtag 追蹤碼位置，正式部署時需將 `G-XXXXXXXXXX` 替換為自己的 Measurement ID。

JavaScript 已規劃三個自訂事件：

1. `click_to_line_inquiry`  
   觸發時機：使用者點擊「LINE 詢價／專人訂購」或「立即 LINE 詢價」。  
   目的：衡量主要 CTA 的點擊意願。

2. `submit_form`  
   觸發時機：使用者送出酒類禮盒推薦諮詢表單。  
   目的：衡量潛在客戶名單與禮盒需求。

3. `scroll_story`  
   觸發時機：使用者滑動到品牌故事區塊。  
   目的：觀察品牌故事是否有被看見，判斷使用者是否只停留在首屏。

## 5. 期末簡報 6 組 Q&A

### Q1｜TA：為什麼選擇 Z 世代與上班族？
A：因為這兩群人都重視生活風格與情緒價值。Z 世代容易被社群內容、包裝與地方特色吸引；上班族則有下班後放鬆與送禮的需求。Hello May 喝樂梅可以同時回應「微醺療癒」與「質感禮盒」兩種情境。

### Q2｜SEO：為什麼關鍵字要放「苗栗梅酒推薦」與「酒類禮盒推薦」？
A：這兩組關鍵字分別對應自飲與送禮需求。「苗栗梅酒推薦」可以吸引正在找地方特色酒款的人；「酒類禮盒推薦」則對應商務饋贈與節慶禮品搜尋。再搭配「苗栗在地酒廠」、「苗栗頭屋名產」，能讓網站更貼近地方品牌定位。

### Q3｜網站：為什麼不直接做一般購物車，而是做 LINE 詢價？
A：酒類商品常涉及數量、配送、年齡提醒、禮盒搭配與到貨時間確認。對地方品牌而言，LINE 詢價比冷冰冰的購物車更適合初期轉換，也能透過專人回覆提高信任感。

### Q4｜數據：GA4 追蹤能幫助什麼？
A：GA4 可以觀察使用者是否點擊 LINE、是否送出表單、是否滑到品牌故事區。這些行為能判斷網站是「有被看見」還是「有帶來詢價」，也能作為期末報告中評估成效的依據。

### Q5｜倫理：酒類網站需要注意什麼？
A：需要明確標示未滿 18 歲禁止飲酒、禁止酒駕、飲酒過量有害健康，並避免用誇大療效或過度鼓勵飲酒的語言。網站中的「微醺療癒」應被理解為生活情境與情緒放鬆，而不是醫療效果。

### Q6｜地方特色：這個網站如何呈現苗栗頭屋的價值？
A：網站不是只把苗栗當產地標籤，而是把頭屋的梅果、東方美人茶、在地發酵工藝與禮盒情境整合成品牌故事。消費者買到的不只是酒，而是一份能被送出與記住的地方文化。

## 6. 倫理、限制與下一步改善

### 資料蒐集透明度
目前表單會蒐集姓名、聯絡方式、用途、預算與補充需求。正式上線時，應加入隱私權聲明，清楚告知資料用途僅作為禮盒諮詢、訂購聯繫與服務改善，不應未經同意轉作其他行銷用途。

### 個人化測驗推薦的偏誤風險
若未來加入「酒類禮盒推薦測驗」，可能因選項設計太少而產生偏誤，例如把所有上班族都推向同一種酒款，或忽略不飲酒者、低酒精需求者與預算限制。因此推薦邏輯應保留「不確定／需要專人建議」選項，並讓使用者可自行修改需求。

### 若 GA4 顯示跳出率高，下一步優化策略
1. 檢查首屏 H1 是否太抽象，必要時加入更直接的商品利益點，例如「苗栗茶梅酒禮盒，LINE 一對一詢價」。
2. 將 LINE CTA 固定在手機底部，降低找不到購買入口的問題。
3. 替換成真實產品照與真實 KOL 截圖，提高信任感。
4. 壓縮圖片、優化載入速度，避免手機端等待時間過長。
5. A/B 測試不同 CTA 文案，例如「找我的禮盒建議」與「請專人幫我配酒」。

## 7. 部署方式

1. 建立 GitHub repository。
2. 上傳 `index.html`、`style.css`、`script.js`、`README.md`。
3. 到 GitHub repository 的 Settings → Pages。
4. Source 選擇 `Deploy from a branch`。
5. Branch 選擇 `main`，資料夾選擇 `/root`。
6. 儲存後等待 GitHub Pages 產生網址。
